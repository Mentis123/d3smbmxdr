Responsible AI Wrap — applied to every layer, every interaction

Governance is not a layer. It's a wrap. Every prompt outbound to a model and every response inbound from a model passes through the same bidirectional control plane — regardless of which consumption mode the customer is on. This is the "never bolted on at the end" principle made operational.

The interop fabric — every seam is an open standard

Four protocols cover the four seams of the stack. Three are open, multi-vendor and in production. The fourth is the lowest-common-denominator fallback that every SDK supports. This is what makes "no vendor lock-in" a diagrammable claim instead of a slogan.
MCP Linux Foundation
Agent ↕ Tools & Knowledge
Model Context Protocol. JSON-RPC 2.0 over stdio or streamable HTTP. Three primitives: Tools, Resources, Prompts. Donated to the Linux Foundation in December 2025 — now vendor-neutral.
Adopted by: Microsoft · Google · AWS · Salesforce · ServiceNow · Anthropic · OpenAI · Databricks
A2A Open proposal
Agent ↔ Agent (peer)
Agent-to-Agent protocol. Capability cards, task delegation, multi-vendor agent choreography. Handles how two agents find each other and collaborate across trust boundaries.
50+ launch partners including Salesforce, SAP, ServiceNow, Workday, Atlassian, Accenture, Deloitte, TCS
AG-UI Emerging
Frontend ↕ Agent
Agent-User Interaction protocol. HTTP + SSE streaming of agent events to any web UI. Shared state, human-in-the-loop approvals, generative UI. Microsoft Agent Framework integrated.
Led by: CopilotKit + Microsoft · early adoption across Agent SDKs
OpenAPI Baseline
Tools fallback
OpenAPI + function calling. The legacy but universal way to expose tools to a model. Every MCP server is effectively a wrapper over this; every vendor SDK supports it natively. The safety net.
Universal. Supported everywhere.

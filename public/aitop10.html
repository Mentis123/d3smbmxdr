Top 10 Enterprise AI Applications — Data Security Assessment

Data management, storage locations, and offshore data transfer analysis for AI applications commonly used within Australian enterprise environments.

Author
Adam Rappaport, AI Product and Solutions
Date
16 February 2026

Executive Summary

This report assesses the top 10 AI applications most likely to be adopted within an Australian enterprise context, focusing on three key questions for each: how the app manages data, where data is stored, and whether data is sent offshore. Findings are contextualised against the Australian Privacy Act 1988 (APP 8 — Cross-border disclosure), IRAP PROTECTED requirements, and practical data sovereignty considerations.

01
Microsoft 365 Copilot
Microsoft Corporation
No Training AU Storage Offshore: Conditional
▼
🔒
How Data is Managed
  • No model training on enterprise data. Microsoft explicitly states: “Prompts, responses, and data accessed through Microsoft Graph are not used to train foundation LLMs.”
  • Copilot interaction data (prompts & responses) stored in a hidden TeamsMessagesData folder within each user’s Exchange Online mailbox.
  • Encryption: AES-256 at rest, TLS 1.2+ in transit, with BitLocker and per-file encryption.
  • Microsoft retains prompt/response data for up to 30 days for service operations; configurable via Microsoft Purview retention policies.
  • Opted out of Azure OpenAI abuse monitoring — no human review of prompts/responses.
  • Users can delete their own Copilot activity history. Admins can use Content Search, eDiscovery, and Teams Export APIs.
Anthropic Subprocessor Alert: As of 7 January 2026, Anthropic (Claude) models are enabled by default for most commercial tenants, including Australia. Anthropic models are excluded from in-country processing commitments. ASD recommends disabling Anthropic until a formal risk assessment is completed. Admins can toggle this in the M365 admin centre.
🏠
Where Data is Stored
  • Azure Australia East (Sydney) and Australia Southeast (Melbourne) regions.
  • Data at rest committed to Australia since March 2024 for tenants provisioned in Australia.
  • In-country LLM processing available since end-2025 under the in-country data processing commitment (covers prompt ingestion, model inference, response generation, telemetry).
  • Advanced Data Residency (ADR) add-on available for contractual data-at-rest commitments.
✈
Offshore Data Transfer
  • Under “normal operations,” processing stays in Australia — but exceptions exist:
  • Capacity constraints/failover: Prompts may route offshore during service disruptions.
  • Anthropic models: Excluded from in-country commitments (data may go offshore if enabled).
  • Web search (Bing): Queries sent to Bing service, may process outside AU.
  • Third-party agents/plugins: Data flows to external services under their own terms.
Recommendation: Disable Anthropic subprocessor, evaluate web search settings, enable ADR, and audit third-party agents for strictest data sovereignty posture.
02
ChatGPT / OpenAI Enterprise
OpenAI
No Training AU Storage (at rest) Offshore: Yes
▼
🔒
How Data is Managed
  • No model training on Enterprise data — contractual commitment (not just a setting).
  • Organisation owns its data. OpenAI states: “Your data always remains confidential, secure, and entirely owned by you.”
  • Configurable retention: minimum 90 days, set by workspace admins. Deleted conversations removed within 30 days.
  • AES-256 at rest, TLS 1.2+ in transit. Enterprise Key Management (EKM) available for customer-controlled encryption keys.
  • SOC 2 Type 2, ISO 27001, ISO 27017, ISO 27018, ISO 27701 certified.
  • Data Processing Addendum (DPA) available for Enterprise customers.
  • API platform offers Zero Data Retention (ZDR) option.
🏠
Where Data is Stored
  • Runs on Microsoft Azure infrastructure.
  • Australia is a supported data residency region for data at rest — customer content (conversations, files, custom GPTs, image generations) can be stored at rest in AU.
  • Data residency setting only applies to new workspaces — existing workspaces cannot be migrated retroactively.
  • No OpenAI-owned data centres in Australia (Stargate facilities are US, Norway, UAE only).
✈
Offshore Data Transfer
Critical: All model inference (prompt processing) occurs on US-based infrastructure. Every prompt is transmitted to US servers for processing, regardless of data residency settings. Inference residency is currently US-only with no announced timeline for Australian availability.
  • Data at rest can be pinned to Australia, but every interaction sends data to the US for processing.
  • Under APP 8, each prompt containing personal information likely constitutes a cross-border disclosure.
  • The Australian entity remains accountable for how OpenAI handles data overseas (s16C Privacy Act).
Alternative: Consider Azure OpenAI Service (a separate Microsoft product) deployed within Australian Azure regions for full data residency, if required.
03
GitHub Copilot
GitHub (Microsoft)
No Training Code in AU (GHEC) Offshore: Yes
▼
🔒
How Data is Managed
  • No training on Business/Enterprise code — default, no opt-out needed.
  • Code editor prompts/suggestions: not retained. Chat/CLI/Mobile: retained 28 days for conversation continuity.
  • Content exclusions available at org and enterprise level to prevent sensitive repos from being processed.
  • Pre-inference screening via GitHub proxy (toxicity, relevance, jailbreak detection). Post-inference checks for code quality, credentials, PII.
  • Third-party model agreements: Anthropic — zero data retention; Google — no training, no prompt logging.
  • User engagement data retained for 2 years.
🏠
Where Data is Stored
  • GHEC with data residency in Australia available since February 2025 (first APAC market). Hosted on Azure Australian regions.
  • Source code repos, PRs, issues, comments stored in-region on dedicated ghe.com subdomain.
  • AI models hosted in GitHub-owned Azure tenants (primarily US), plus AWS (Anthropic) and GCP (Google).
✈
Offshore Data Transfer
Critical: GitHub states: “Traffic is routed to regions based on capacity; GitHub cannot guarantee any traffic will stay within a single region.” AI inference is not covered by data residency — code context is sent offshore for processing.
  • Code at rest stays in Australia (with GHEC data residency). All Copilot AI processing goes offshore (likely US).
  • Telemetry data may also be processed outside Australia.
  • This is a split architecture: storage is in-region, inference is not.
  • No public timeline for in-region AI inference (unlike M365 Copilot which received this commitment separately).
04
ServiceNow Now Assist
ServiceNow
Opt-out Training AU Storage Offshore: Conditional
▼
🔒
How Data is Managed
  • Stateless in-memory processing — prompts/responses processed in memory and immediately discarded. No persistent retention of AI processing data.
  • Data sharing for model improvement: opt-out available via Now Assist Admin Console. Only anonymised/masked data used if opted in. Opting out does not degrade functionality.
  • Only minimum necessary data leaves the instance (input prompt + related fields).
  • Data Privacy for Now Assist plugin available for PII masking (two-way: masks before LLM, restores after). Not enabled by default — must be configured.
  • Logically isolated per-tenant environment prevents cross-customer data exposure.
🏠
Where Data is Stored
  • Two data centres in Australia: Sydney and Melbourne (paired configuration).
  • ServiceNow Protected Platform Australia (SPP AU): hosted exclusively on Azure AU, IRAP PROTECTED assessed (Feb 2025). All data in-country, all support staff AU-based.
  • Now LLM (ServiceNow’s own models) processed in “same region” data centres or Azure within region.
  • AI Agents defaults to GPT-4.1 on ServiceNow-managed Azure servers — specific region unclear.
✈
Offshore Data Transfer
  • Now LLM: Processing stated as “same region” — verify whether this means Australia specifically or broader APAC.
  • SPP AU: All data in-country (strongest guarantee). Confirm AI features availability on this tier.
  • Third-party/BYOM: Data leaves Australia. ServiceNow cannot guarantee visibility into third-party LLM data handling.
  • AI Agents (GPT-4.1): Azure region unconfirmed — clarify with ServiceNow.
  • Historical concern: ServiceNow previously had “incidental transfers” via offshore support staff (addressed by SPP AU).
Recommendation: Use SPP AU for strict sovereignty. Opt out of data sharing. Enable PII masking plugin. Avoid BYOM for regulated workloads. Clarify “same region” definition with ServiceNow directly.
05
Google Gemini
Google / Alphabet
Consumer: Yes / Ent: No No AU Processing Offshore: Yes
▼
🔒
How Data is Managed
  • Consumer Gemini: Data used for training by default. Human reviewers assess a subset of chats (retained up to 3 years). Opt-out available via “Gemini Apps Activity” settings.
  • Workspace Enterprise: NOT used for training, not reviewed by humans, not saved beyond session.
  • Vertex AI (Cloud): Not used for training. 24hr in-memory cache by default. Zero Data Retention (ZDR) available.
  • Compliance: ISO 42001, HITRUST, PCI-DSS v4.0. Vertex AI ZDR requires disabling caching and abuse monitoring per-project.
🏠
Where Data is Stored
  • Google Cloud has australia-southeast1 (Sydney) and australia-southeast2 (Melbourne) regions.
  • BUT: Gemini model serving and formal data residency guarantees do NOT include Australia.
  • Workspace Gemini: data regions limited to US and EU only. No Australian option.
  • Vertex AI: AU endpoint exists but no formal ML processing guarantee for Australia. DRZ only supported in US/EU.
  • Assured Workloads Australia Data Boundary available but ML inference guarantee unclear.
✈
Offshore Data Transfer
Critical: For Workspace Gemini, processing is confined to US or EU with no Australian option. For Vertex AI, Google provides no formal guarantee ML processing stays in Australia even when using the AU endpoint.
  • Consumer Gemini: processed in US/UK/EEA/Singapore.
  • Workspace Gemini: US/EU only processing — all AU enterprise prompts go offshore.
  • Vertex AI: AU endpoint exists but “no guarantee ML processing occurs at a specific location” for unlisted regions.
Mitigation: Use Vertex AI with Assured Workloads AU boundary + ZDR. Implement client-side encryption for Workspace. Enforce prompt hygiene policies to prevent PII in Gemini prompts.
06
Anthropic Claude
Anthropic
No Training US Storage (direct) Offshore: Conditional
▼
🔒
How Data is Managed
  • No training on commercial data (Team/Enterprise/API plans) — contractual commitment.
  • API retention: 7 days (reduced from 30). Zero Data Retention (ZDR) available for enterprise API.
  • AES-256 at rest, TLS 1.2+ in transit.
  • SOC 2 Type I/II, ISO 27001, ISO 42001 (AI Management System) certified.
  • Free/consumer tier: conversations may be used for safety research/training (opt-out available).
🏠
Where Data is Stored
  • Direct API / claude.ai: All data stored in US. No Australian data residency option for direct API.
  • AWS Bedrock ap-southeast-2 (Sydney): Claude models available for inference in Australia. Claude 3.5 Sonnet and Claude 3 Haiku available in Sydney region. IRAP PROTECTED certified.
  • Multi-region processing introduced Aug 2025 but storage remains US for direct API.
Key Finding: AWS Bedrock in Sydney (ap-southeast-2) is the recommended path for Australian data residency with Claude. Inference can stay in Australia via Bedrock, with IRAP PROTECTED certification.
✈
Offshore Data Transfer
  • Direct API / claude.ai: Yes — all data goes to US.
  • AWS Bedrock (Sydney): Inference can stay in Australia. This is the only path to keep Claude data onshore.
  • Data#3 as a Microsoft-centric partner should evaluate AWS Bedrock integration requirements.
Recommendation: Use AWS Bedrock in ap-southeast-2 (Sydney), NOT direct API, for Australian data sovereignty. Avoid claude.ai for enterprise workloads requiring AU residency.
07
Salesforce Einstein AI / Agentforce
Salesforce
No Training AU Storage Offshore: Conditional
▼
🔒
How Data is Managed
  • Einstein Trust Layer: PII masking before LLM processing, toxicity detection, zero data retention with third-party LLMs.
  • Data stays within Salesforce VPC. Default model: OpenAI GPT-4o.
  • Trust Layer performs dynamic grounding, prompt defence, and audit trail generation.
  • No enterprise customer data used for model training.
🏠
Where Data is Stored
  • Salesforce Hyperforce on AWS Sydney available for Australian customers.
  • Geo-aware routing based on Data Cloud region. Global data residency options include Australia.
  • Sovereign AI approach available for Australian requirements.
✈
Offshore Data Transfer
  • Geo-aware routing attempts to keep data in-region.
  • Fallback risk: “If a model isn’t available in your Data Cloud region, or if there isn’t enough capacity, requests are routed to the US.”
  • InCountry AgentCloak available for strict sovereignty needs (additional product).
Recommendation: Deploy on Hyperforce AU. Monitor geo-aware routing behaviour. Evaluate InCountry AgentCloak for strictest requirements. Confirm model availability in AU Data Cloud region.
08
Zoom AI Companion
Zoom Video Communications
No Training AU Meetings Offshore: Yes (default)
▼
🔒
How Data is Managed
  • No customer content used for AI training (ToS Section 10). Explicit commitment.
  • Federated AI architecture using Zoom SLMs + third-party LLMs (OpenAI, Anthropic, Perplexity).
  • Three deployment models: Federated (default, full features), ZM+ (Zoom + Anthropic on Bedrock), ZMO (Zoom-only models, reduced features).
  • AES-256 at rest, TLS 1.2 in transit. Zero data retention option available.
🏠
Where Data is Stored
  • Zoom data centres in Sydney and Melbourne for meeting data.
  • AI processing location depends on deployment model chosen.
  • Third-party providers (OpenAI, Anthropic) may retain data in US for up to 30 days.
✈
Offshore Data Transfer
Default (Federated) mode sends data to US third-party providers. Custom AI Companion is explicitly unavailable for Australia-hosted accounts.
  • Federated (default): Yes — data sent to US third-party providers (OpenAI, Anthropic, Perplexity).
  • ZMO (Zoom-only): Could keep processing in-region but Australia NOT explicitly listed as confirmed ZMO region (US, Europe, Canada, India confirmed).
  • Third-party providers retain data in US up to 30 days.
Recommendation: Evaluate ZMO deployment for reduced offshore exposure. Confirm ZMO regional availability for Australia with Zoom directly. Accept reduced feature set if ZMO is selected.
09
Adobe Firefly
Adobe Inc.
No Training US Only Storage Offshore: Yes
▼
🔒
How Data is Managed
  • Never trains on enterprise customer data.
  • IP indemnification available for qualifying enterprise plans.
  • Content Credentials (C2PA) system for provenance tracking on generated content.
  • Partner models (OpenAI, Google) can be disabled by admins.
🏠
Where Data is Stored
  • AI inference: ONLY AWS US-East and US-West regardless of user location.
  • Generation History: US-East, EMEA-West, or APAC-East (Japan, not Australia).
  • Content Credentials stored in US-East only.
  • No Australian data centres for Firefly. Edge servers in AU for CDN only, not processing.
✈
Offshore Data Transfer
ALL processing occurs in the US. No Australian option whatsoever. This is the strongest “offshore” finding across all 10 applications assessed.
  • Every prompt, image generation request, and output is processed in US data centres.
  • No data residency options for Australia. No public roadmap for AU expansion.
  • Content Credentials metadata stored exclusively in US-East.
Risk Assessment: Suitable for non-sensitive creative work. Avoid for workloads involving confidential data, personal information, or content subject to Australian data sovereignty requirements.
10
Perplexity AI
Perplexity AI Inc.
No Training (Ent.) US Only Storage Offshore: Yes
▼
🔒
How Data is Managed
  • Enterprise Pro: No training on data — contractual. Zero data retention with all third-party model providers.
  • Files auto-deleted after 7 days. Custom retention for organisations with 50+ seats.
  • SOC 2 Type II, HIPAA (with BAA) certified.
🏠
Where Data is Stored
  • All infrastructure on AWS (US). New $750M Azure deal announced January 2026.
  • Servers in US and EU only. No APAC hosting.
  • No Australian data residency. No public commitments for future AU residency.
✈
Offshore Data Transfer
ALL data processed in the US. No Australian data residency, hosting, or processing options. DPA lacks Australia-specific provisions.
  • Every query and response is processed on US infrastructure.
  • No regional hosting options for Australia.
  • Suitable for general research queries not involving sensitive/personal data.
